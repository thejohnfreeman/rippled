3 ## Specifying Files
4 
5 To specify the source files for which to build documentation, modify `INPUT`
6 and its related fields in `docs/Doxyfile`. Note that the `INPUT` paths are
7 relative to the `docs/` directory.
8 
9 ## Install Dependencies
10 
11 ### Windows
12 
13 Install these dependencies:
14 
15 1. Install [Doxygen](http://www.stack.nl/~dimitri/doxygen/download.html)
16 
17 ### MacOS
18 
19 1. Install doxygen:
20  * Use homebrew to install: `brew install doxygen`. The executable will be
21  installed in `/usr/local/bin` which is already in your path.
22  * Alternatively, install from here: [doxygen](http://www.stack.nl/~dimitri/doxygen/download.html).
23  You'll then need to make doxygen available to your command line. You can
24  do this by adding a symbolic link from `/usr/local/bin` to the doxygen
25  executable. For example, `$ ln -s /Applications/Doxygen.app/Contents/Resources/doxygen /usr/local/bin/doxygen`
26 
27 ### Linux
28 
29 1. Install doxygen using your package manager OR from source using the links above.
30 
31 ### [Optional] Install Plantuml (all platforms)
32 
33 Doxygen supports the optional use of [plantuml](http://plantuml.com) to
34 generate diagrams from `@startuml` sections. We don't currently rely on this
35 functionality for docs, so it's largely optional. Requirements:
36 
37 1. Download/install a functioning java runtime, if you don't already have one.
38 2. Download [plantuml](http://plantuml.com) from
39  [here](http://sourceforge.net/projects/plantuml/files/plantuml.jar/download).
40  Set a system environment variable named `PLANTUML_JAR` with a value of the fullpath
41  to the file system location of the `plantuml.jar` file you downloaded.
42 
43 ## Do it
44 
45 ### all platforms
46 
47 From the rippled root folder:
48 ```
49 cd docs
50 mkdir -p html_doc
51 doxygen Doxyfile
52 ```
53 The output will be in `docs/html_doc`.
54 
55 ## Docker
56 
57 (applicable to all platforms)
58 
59 Instead of installing the doxygen tools locally, you can use the provided `Dockerfile` to create
60 an ubuntu based image for running the tools:
61 
62 1. Install [Docker](https://docs.docker.com/engine/installation/)
63 2. Build Docker image. From the rippled root folder:
64 
65 ```
66 sudo docker build -t rippled-docs docs/
67 ```
68 
69 Then to run the image, from the rippled root folder:
70 
71 ```
72 sudo docker run -v $PWD:/opt/rippled --rm rippled-docs
73 ```
74 
75 The output will be in `docs/html_doc`.

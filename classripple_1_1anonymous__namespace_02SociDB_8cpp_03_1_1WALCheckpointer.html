Detailed Description

Run a thread to checkpoint the write ahead log (wal) for the given soci::session every 1000 pages.

This is only implemented for sqlite databases.

Note: According to: https://www.sqlite.org/wal.html#ckpt this is the default behavior of sqlite. We may be able to remove this class.

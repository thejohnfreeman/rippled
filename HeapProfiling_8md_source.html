1 ## Heap profiling of rippled with jemalloc
2 
3 The jemalloc library provides a good API for doing heap analysis,
4 including a mechanism to dump a description of the heap from within the
5 running application via a function call. Details on how to perform this
6 activity in general, as well as how to acquire the software, are available on
7 the jemalloc site:
8 [https://github.com/jemalloc/jemalloc/wiki/Use-Case:-Heap-Profiling](https://github.com/jemalloc/jemalloc/wiki/Use-Case:-Heap-Profiling)
9 
10 jemalloc is acquired separately from rippled, and is not affiliated
11 with Ripple Labs. If you compile and install jemalloc from the
12 source release with default options, it will install the library and header
13 under `/usr/local/lib` and `/usr/local/include`, respectively. Heap
14 profiling has been tested with rippled on a Linux platform. It should
15 work on platforms on which both rippled and jemalloc are available.
16 
17 To link rippled with jemalloc, the argument
18 `profile-jemalloc=<jemalloc_dir>` is provided after the optional target.
19 The `<jemalloc_dir>` argument should be the same as that of the
20 `--prefix` parameter passed to the jemalloc configure script when building.
21 
22 ## Examples:
23 
24 Build rippled with jemalloc library under /usr/local/lib and
25 header under /usr/local/include:
26 
27  $ scons profile-jemalloc=/usr/local
28 
29 Build rippled using clang with the jemalloc library under /opt/local/lib
30 and header under /opt/local/include:
31 
32  $ scons clang profile-jemalloc=/opt/local
33

36 ## Using the jemalloc library from within the code
37 
38 The `profile-jemalloc` parameter enables a macro definition called
39 `PROFILE_JEMALLOC`. Include the jemalloc header file as
40 well as the api call(s) that you wish to make within preprocessor
41 conditional groups, such as:
42 
43 In global scope:
44 
45  #ifdef PROFILE_JEMALLOC
46  #include <jemalloc/jemalloc.h>
47  #endif
48 
49 And later, within a function scope:
50 
51  #ifdef PROFILE_JEMALLOC
52  mallctl("prof.dump", NULL, NULL, NULL, 0);
53  #endif
54 
55 Fuller descriptions of how to acquire and use jemalloc's api to do memory
56 analysis are available at the [jemalloc
57 site.](http://www.canonware.com/jemalloc/)
58 
59 Linking against the jemalloc library will override
60 the system's default `malloc()` and related functions with jemalloc's
61 implementation. This is the case even if the code is not instrumented
62 to use jemalloc's specific API.
